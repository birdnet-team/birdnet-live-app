# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in BirdNET Live, please report it responsibly.

**Do NOT open a public issue for security vulnerabilities.**

Instead, please email: **ccb-birdnet@cornell.edu**

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix release**: As soon as practical, typically within 30 days

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| Latest  | Yes                |
| Older   | Security fixes only |

## Scope

This security policy covers:
- The BirdNET Live Flutter application
- Associated build and distribution infrastructure
- Documentation site

Out of scope:
- The BirdNET model itself (report to Cornell Lab)
- Third-party dependencies (report to respective maintainers)
